# CDD Agent – Adverse Media Screening Demo

A lightweight AI-powered demo for **Customer Due Diligence (CDD)** and **Adverse Media Screening**.

This application uses a Large Language Model (LLM) with web search capabilities to identify and summarise potential red flags related to individuals or companies. The tool is designed as a **proof-of-concept for compliance, AML, and risk investigation workflows**.

The interface is built with **Streamlit**, allowing analysts to run screening queries and generate structured summaries that can be exported into a report.

---

# Features

- Adverse media search using LLM + web search
- Automated summarisation of relevant news stories
- Red flag classification based on predefined compliance criteria
- Interactive Streamlit dashboard
- Export selected findings into a structured report (.docx)
- Local history tracking of screening results

# Installation

## 1. Clone the repository
git clone https://github.com/amazingclaude/cdd_agent.git

cd cdd_agent


---

## 2. Create the Conda environment
conda env create -f environment.yml
conda activate cdd_agent


If the environment file fails for any reason, install dependencies manually:

pip install -r requirements.txt


---

# Setting the OpenAI API Key

This application requires an **OpenAI API key** to run the LLM-based screening.

You can configure it locally using one of the following methods.

---

## Option 1 – Environment Variable (Recommended)

### macOS / Linux
export OPENAI_API_KEY="your_openai_api_key"


### Windows PowerShell

After setting the variable, restart the terminal.

---

## Option 2 – `.env` File

Create a file named `.env` in the project root:

from dotenv import load_dotenv
load_dotenv()


⚠️ Never commit `.env` files to GitHub.

Ensure `.env` is included in `.gitignore`.

---

# Running the Application

Start the Streamlit app:

streamlit run app.py


Streamlit will launch automatically.

If not, open the browser and go to:
http://localhost:8501

# Example Use Case

Example screening query:

Entity Name: Wirecard
Country: Germany
Sector: Financial Services


The system will:

1. Search reputable online sources
2. Identify potentially adverse news
3. Summarise findings
4. Tag relevant red flags
5. Allow analysts to export selected items into a report

---

# Red Flag Criteria

The screening logic evaluates news findings against predefined compliance categories such as:

- Fraud and financial crime
- Corruption and bribery
- Regulatory enforcement
- Sanctions exposure
- Litigation and investigations

These criteria are configurable in:
criteria_config.py


---

# Output

Selected findings can be exported into a **Word report (.docx)** containing:

- entity information
- screening summary
- red flag classifications
- supporting sources

This mirrors typical outputs used in **CDD / AML investigation workflows**.
